# BOSSA AI OS — Operating Rules

## Rule 1 — No duplicate storage
Notion pages should route and review.
Source data should live in the proper databases.

## Rule 2 — No vague decisions
Every meaningful decision should include:
- what was decided
- why now
- who owns it
- when it will be reviewed

## Rule 3 — Weekly rhythm beats random effort
A simple weekly cycle is better than random bursts of activity.

## Rule 4 — Freshness matters
Old data weakens the system.
Competitor reviews, snapshots, and promos must be updated consistently.

## Rule 5 — Evidence beats assumptions
Whenever possible, claims should be supported by:
- screenshots
- links
- menu captures
- Instagram promo evidence

## Rule 6 — AI supports operations
ChatGPT, Claude, and other models should improve clarity and speed, not create chaos.

## Rule 7 — Repo is the source of truth for files
- Notion = workspace
- Repo = durable file system
- VS Code = build environment

## Rule 8 — Keep the system simple
If a page, prompt, or file does not help BOSSA decide or execute better, simplify it.
